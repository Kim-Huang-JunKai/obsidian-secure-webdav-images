# 更新日志

## 0.0.32 — 签名体系重构与完整同步修复

### 签名体系重构

文件签名从 MD5 哈希改为基于时间戳的三元组 `{ctime}:{mtime}:{size}`：

- **ctime（创建时间）**：标识文件身份，区分同名不同文件
- **mtime（修改时间）**：检测文件变化
- **size（文件大小）**：辅助校验

本地和远端使用统一的签名格式，消除了之前本地用哈希、远端用时间戳的不一致问题。

### 墓碑机制增强

- 墓碑记录 `creationTime` 字段，删除时与远端/本地文件的 `ctime` 匹配
- 防止同名不同文件被误删（`isTombstoneAuthoritative` 先检查 creationTime）
- `shouldDeleteLocalFromTombstone` 用 creationTime 判断是否同一文件，5 秒容差

### 完整同步修复

- **恢复孤立索引清理**：重新接入 `reconcileOrphanedSyncEntries`，处理索引中存在但本地已删除的路径（如重命名后的旧路径）
- **远端对账增加墓碑检查**：`reconcileRemoteAndLocal` 在下载远端文件前检查墓碑，避免已删除文件被恢复

### 快速同步与完整同步策略说明

| | 快速同步 | 完整同步 |
|---|---|---|
| 扫描范围 | 队列中的文件 + 远端事件 | 全量扫描远端 + 本地所有文件 |
| 远端请求量 | 少量 | 大量 |
| 适用场景 | 日常高频操作 | 首次同步、定期同步、手动触发 |
| 删除处理 | 队列驱动，30 秒宽限期 | 全局墓碑扫描 + 孤立索引清理 |
| 冲突检测 | 不检测 | 完整双向签名比较 + 冲突副本 |
