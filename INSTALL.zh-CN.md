# Secure WebDAV Images 中文安装与使用说明

## 插件定位

`Secure WebDAV Images` 的核心目标是：

- 把图片从笔记中剥离到独立的 WebDAV 图片目录
- 把笔记和非图片附件同步到另一套远程笔记目录
- 在移动端尽量减少本地存储压力
- 避免笔记继续依赖网页外链图片

这个插件不是“公开图床插件”，也不是“Git 式精确合并同步器”。  
它更适合这类用户：

- 图片很多，移动端越来越重
- 希望图片和笔记分目录管理
- 愿意用 WebDAV 作为自己的私有远端存储
- 能接受先手动验证，再逐步开启自动同步和懒加载

## 安装前请先了解

### 1. 存储机制

插件使用两套远程目录：

- `图片远程目录`
  只保存图片。
- `远程笔记目录`
  保存 Markdown 笔记和非图片附件。

### 2. 懒加载模式的真实含义

当你把 `笔记本地保留模式` 设为 `按需加载笔记` 时：

- 服务器保存的是完整笔记正文
- 本地设备保存的是目录结构和占位文件
- 占位文件只是“本地减量缓存方案”，不是远端正文

### 3. 重要警告

如果你不能确认远程 WebDAV 的稳定性，请先不要启用 `按需加载笔记`。

建议先这样使用：

1. 先用 `完整本地`
2. 先手动同步一段时间
3. 确认远端正文始终完整可恢复
4. 再决定是否切到 `按需加载笔记`

## 0.0.8 最近修复的关键点

这一版重点修复的是懒加载和删除同步的安全问题：

- 懒加载占位文件不会再被当成正文上传到服务器
- 删除传播不再只靠时间戳推断，而是使用明确的删除墓碑
- 本地后来又修改过的笔记，不会被旧删除记录误删
- 本地回收为占位前，会先校验远端正文是否真的可读
- 如果远端正文连续确认缺失，占位文件会自动清理，不会长期假装“还能恢复”

这意味着当前版本追求的是：

- 服务器尽量保留完整正文
- 本地懒加载只做减量，不反向清空服务器
- 失效占位不会无限残留

## 桌面端安装

推荐直接下载发布包：

```text
secure-webdav-images-0.0.8.zip
```

安装步骤：

1. 下载压缩包。
2. 解压压缩包。
3. 打开你的 vault 目录。
4. 进入：

```text
.obsidian/plugins/secure-webdav-images/
```

5. 如果目录不存在，就手动创建。
6. 把以下文件复制进去：
   - `main.js`
   - `manifest.json`
   - `styles.css`
7. 重启 Obsidian。
8. 进入 `设置 > 社区插件`，启用 `Secure WebDAV Images`。

## 手机端安装

### Android

1. 下载：

```text
secure-webdav-images-0.0.8.zip
```

2. 解压压缩包。
3. 找到你的 vault 目录。
4. 进入：

```text
<你的 Vault>/.obsidian/plugins/secure-webdav-images/
```

5. 如果目录不存在，就手动创建。
6. 把以下文件复制进去：
   - `main.js`
   - `manifest.json`
   - `styles.css`
7. 重启 Obsidian。
8. 在 `设置 > 社区插件` 中启用插件。

更新插件时，建议直接覆盖：

- `main.js`
- `manifest.json`
- `styles.css`

不要删除：

- `data.json`

这样可以保留你的配置项。

### iPhone / iPad

iOS 手动安装插件会更麻烦，因为文件管理和沙盒限制更严格。  
最稳妥的方式是先在桌面端安装到同一个 vault，再通过你的 vault 同步方式把插件目录带到移动端。

## 首次配置

启用插件后，先配置这些项目：

- `WebDAV 基础地址`
- `账号`
- `密码`
- `图片远程目录`
- `远程笔记目录`
- `自动同步频率`
- `笔记本地保留模式`
- `笔记本地回收天数`

建议：

- 图片目录和笔记目录必须分开
- 不要把两者配置到同一个远端目录

示例：

```text
WebDAV 基础地址: https://your-webdav-host
图片远程目录: remote-images
远程笔记目录: remote-notes
```

## 安装后建议的验证顺序

### 1. 先测试 WebDAV

运行命令：

```text
Test WebDAV connection
```

它会测试：

- 上传
- 读取
- 删除

### 2. 再迁移旧笔记图片

如果你的旧笔记里还是原生图片引用：

- `![](...)`
- `![[...]]`

可以在设置页运行一次性工具，把它们迁移到新的远程图片模式。

### 3. 再执行笔记同步

运行命令：

```text
Sync vault content to WebDAV
```

### 4. 最后再考虑懒加载

只有在下面这些都确认稳定之后，才建议启用 `按需加载笔记`：

- 远端笔记目录结构正确
- 远端正文能正常回读
- 多设备同步结果稳定

## 命令说明

### `Upload local images in current note to WebDAV`

作用：

- 只处理当前笔记里的本地图片
- 上传图片到远端图片目录
- 把笔记改写成 `secure-webdav` 代码块
- 上传成功后删除本地图片

### `Test WebDAV connection`

作用：

- 快速验证 WebDAV 的上传、读取、删除链路是否可用

### `Sync vault content to WebDAV`

作用：

- 对账同步笔记和非图片附件
- 上传新增内容
- 拉取远端更新
- 删除明确已删除的远端笔记内容
- 安全地处理懒加载占位和删除墓碑

## 懒加载模式的明确说明

### 会发生什么

- 长时间未访问的 Markdown 笔记，可能被替换成本地占位文件
- 真正打开时，再从远端笔记目录恢复完整正文

### 不会发生什么

- 占位文件不会被当成正文上传
- 占位文件不会拿去覆盖服务器上的完整笔记

### 远端正文丢失时怎么处理

插件采用两阶段策略：

1. 第一次确认远端缺失：
   先保留本地占位，防止临时网络异常造成误判。
2. 再次确认远端仍缺失：
   自动清理本地失效占位。

所以，占位文件不是永久保留的。

## 不适合安装的人群

如果你属于下面任一情况，建议谨慎使用，或只使用 `完整本地` 模式：

- 远端 WebDAV 服务不稳定
- 设备间经常离线很久后再同时修改
- 你更需要 Git 式精细合并，而不是对账同步
- 你无法接受任何“先保守保留、再次确认后清理”的同步策略
